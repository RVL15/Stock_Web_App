STEP 2: PREPARE YOUR CODE (2 MINUTES)
======================================

Create 2 files in your project folder:

FILE 1: web.config
Location: C:\Users\Admin\Desktop\stock app 1\web.config

Content (copy-paste):
---
<?xml version="1.0" encoding="utf-8"?>
<configuration>
  <system.webServer>
    <handlers>
      <add name="PythonHandler" path="*" verb="*" 
           modules="FastCgiModule" 
           scriptProcessor="D:\home\python\python.exe|D:\home\python\wfastcgi.py" 
           resourceType="Unspecified" 
           requireAccess="Script" />
    </handlers>
  </system.webServer>
</configuration>
---

FILE 2: .deployment
Location: C:\Users\Admin\Desktop\stock app 1\.deployment

Content (copy-paste):
---
[config]
command = pip install -r requirements.txt
---

✓ Both files created? Move to STEP 3


STEP 3: PUSH CODE TO GITHUB (3 MINUTES)
========================================

Your code is already in Git (I saw "git push" earlier).

In PowerShell:

cd "C:\Users\Admin\Desktop\stock app 1"

git add .

git commit -m "Add Azure deployment files"

git push origin main

¿ Code pushed to GitHub? Move to STEP 4


STEP 4: CREATE AZURE APP SERVICE (5 MINUTES)
==============================================

1. Go to Azure Portal:
   https://portal.azure.com

2. Click blue "+ Create a resource" button

3. Search for: "App Service"

4. Click "App Service" → Click "Create"

5. Fill in these details:

   Project details:
   - Subscription: (your subscription - auto-selected)
   - Resource Group: 
     ✓ Create new → Name: "stock-rg"

   Instance details:
   - Name: "stock-dashboard-YOUR-NAME"
     (must be unique, e.g., "stock-dashboard-john")
   - Publish: Code
   - Runtime stack: Python 3.11
   - OS: Linux
   - Region: West US 2 (or closest to you)

   App Service Plan:
   - Sku and size: Free F1 (free tier)
     (Click "Change size" if needed)

6. Click "Review + Create"

7. Click "Create" and wait 2-3 minutes

8. When done, click "Go to resource"

✓ App Service created? Move to STEP 5


STEP 5: CONNECT GITHUB & DEPLOY (2 MINUTES)
============================================

1. In App Service, left menu click:
   "Deployment" → "Deployment Center"

2. Source: Select "GitHub"

3. Click "Authorize"
   (Browser opens to GitHub login)

4. Select your GitHub account and authorize

5. Fill in:
   - Organization: your-github-username
   - Repository: stock app 1 (or your repo name)
   - Branch: main

6. Click "Save"

Azure now builds and deploys automatically!

Wait 2-3 minutes...

TROUBLESHOOTING
===============

Problem: Shows error page or 502 Bad Gateway

Solution:
1. Go to Azure Portal
2. App Service → Monitor → Log Stream
3. Look for red error messages
4. Common issues:

   "ModuleNotFoundError: Flask"
   → Flask wasn't installed
   → Click Console (top)
   → Run: pip install -r requirements.txt
   → Restart app

   "No such file or directory: app.py"
   → app.py isn't in root
   → Check on GitHub in correct folder

   "Connection timeout to NSE/Yahoo"
   → App can't access stock data
   → Check internet connectivity
   → Try different stock endpoint


Problem: CSS/JS not loading (dashboard looks broken)

Solution:
1. Hard refresh: CTRL + F5 (Windows) or CMD + SHIFT + R (Mac)
2. Check browser console (F12) for errors
3. Verify static/ folder on GitHub
4. Check file permissions in Azure Console


Problem: API returns empty data

Solution:
1. Check NSE/Yahoo Finance are up
2. Increase timeout: Azure Portal → Configuration → General settings
3. Test locally first: python app.py
4. Check Log Stream for errors


Problem: Can't access Azure Portal

Solution:
1. Clear browser cache (CTRL + SHIFT + Delete)
2. Try different browser
3. Logout and login again
4. Check account isn't locked

UPGRADE TO PAID TIER
====================

Free tier limitations:
- 60 minutes/day
- 1 GB memory
- Shared infrastructure

Upgrade when needed:

1. Azure Portal → App Service
2. Settings → App Service plan
3. Click "Change size"
4. Select Basic B1 (~$12.50/month)
5. Review and update

Paid tier benefits:
- 24/7 uptime
- Better performance
- More resources
- SSL certificates

SECURITY
========

Best practices:

1. Secrets:
   - Never put API keys in code
   - Use Azure Key Vault
   - Set as environment variables

2. HTTPS:
   - Automatic (Azure handles it)
   - Delete HTTP version (redirect to HTTPS)

3. Firewall:
   - Restrict IPs if needed
   - Networking → Inbound rules

4. Backups:
   - Set up automated backups
   - Backup → Configure
